# MemoryNode

MemoryNode is a multi-tenant, API-first memory service delivered as a single Cloudflare Worker. It provides a REST API, a hosted Model Context Protocol (MCP) JSON-RPC endpoint, and a session-cookie-authenticated browser console — all backed by Supabase Postgres with pgvector for embeddings and PayU for billing.

## Production surface

| Component | URL | Source |
| --- | --- | --- |
| REST API | `https://api.memorynode.ai` | [apps/api](apps/api/) |
| Hosted MCP | `https://mcp.memorynode.ai` | [apps/api/src/mcpHosted.ts](apps/api/src/mcpHosted.ts) |
| Console (billing/admin) | `https://console.memorynode.ai` | [apps/dashboard](apps/dashboard/) |
| App (end-user) | `https://app.memorynode.ai` | [apps/dashboard](apps/dashboard/) |
| Data plane | Supabase Postgres + pgvector | [infra/sql](infra/sql/) |

Billing is **PayU-only**. Stripe billing is retired; `POST /v1/billing/portal` always returns HTTP 410.

## Monorepo layout

```
apps/
  api/              Cloudflare Worker (memorynode-api)
  dashboard/        Vite React app (memorynode-console + memorynode-app Pages)
packages/
  shared/           Plan definitions, error types, MCP policy engine
  sdk/              @memorynodeai/sdk
  mcp-core/         Hosted MCP tool registry
  mcp-server/       Local stdio MCP binary (memorynode-mcp)
  cli/              Operator CLI
infra/sql/          Postgres migrations
```

## Local development

Requires `pnpm@10`, Node.js 20, and `wrangler` for Worker runs.

```bash
pnpm install
pnpm build                  # build all packages
pnpm test                   # unit tests
pnpm dev                    # run the Worker against a local Supabase (see apps/api/README.md)
pnpm openapi:gen            # regenerate docs/external/openapi.yaml
```

## CI gates

Every PR runs (among others):

- `pnpm check:docs-drift` — prevents doc divergence from truth files.
- `pnpm openapi:check` — OpenAPI matches code.
- `pnpm check:docs-billing` — billing docs remain PayU-only.
- `pnpm check:runbooks` — [INCIDENT_RUNBOOKS.md](docs/internal/INCIDENT_RUNBOOKS.md) retains its 10 required sections.
- `pnpm check:observability-contracts` — alert ids and SLO services match code.
- `pnpm check:least-privilege` — `rls-first` roadmap and migrations intact.
- `pnpm migrations:check` — [docs/internal/README.md](docs/internal/README.md) migration tokens stay in sync.
